# ⚔️ Card Battle TCG

ポケモンカードゲーム風のオリジナル・トレーディングカードゲーム（TCG）の**ブラウザ対戦アプリ**。
ビルド不要のバニラ JavaScript（ES Modules）で動作し、シンプルな AI と 1 人で対戦できます。

> 本物のポケモンの名称・イラスト・カードデータは使用していません。
> すべてオリジナル名＋汎用エレメント（炎/水/草/雷/超/闘/無）のプレースホルダーです。

## 遊ぶ

GitHub Pages: <https://tr-hirama.github.io/card-battle-tcg/>

ローカルで動かす場合は、ES Modules のため **HTTP 配信**が必要です（`file://` では動きません）。

```sh
# 任意の静的サーバでルートを配信して index.html を開く
npx serve .
# または Python: python -m http.server
```

## ルール（v1）

- デッキ 60 枚・初期手札 7 枚・サイド 6 枚・マリガンあり
- 先攻・後攻はランダム。**先攻の最初のターンはドロー・ワザ・サポート不可**。**お互いの最初の番は進化できない**
- 毎ターン：1 ドロー → ベンチ展開／進化／エネルギー 1 枚／トレーナー／にげる → **ワザ（使うとターン終了）**
- エネルギー付け・進化は**手札からポケモンへ**、たねポケモンは**ベンチへ**ドラッグ&ドロップできます（クリック操作でも可）
- 弱点 ×2・抵抗力 −30
- 特殊状態：どく／やけど／ねむり／マヒ／こんらん（ターン間にチェック）
- 相手をきぜつさせるとサイドを 1 枚取る。**サイド 6 枚を取りきれば勝ち**（ほかに山札切れ・場のポケモン全滅でも決着）

## 実カード画像（ローカル限定・任意）

著作権に配慮し、公式カードの画像・データは**リポジトリに一切含めません**。
代わりに、**自分のローカル環境でだけ**、pokemon-card.com の番号で実カード（画像・ステータス）を使える仕組みを用意しています。公開サイト（GitHub Pages）には鍵ファイルが無いため、常にオリジナルのプレースホルダーカードで動作します。

### 仕組み
1. ローカルに鍵ファイル `key.local.js` を置く（`.gitignore` 済み）:
   ```js
   window.__CARD_KEY = 'あなたのパスワード';
   ```
2. 起動時、このパスワードの SHA-256 が `js/config.js` の `IMAGE_UNLOCK_HASH` と一致した時だけ、実カードデータと公式画像が有効になります。
   - パスワードの変更: 新しいパスワードの SHA-256 を計算して `IMAGE_UNLOCK_HASH` を差し替え。
     ```powershell
     $pw="新パスワード"; $s=[Security.Cryptography.SHA256]::Create()
     (($s.ComputeHash([Text.Encoding]::UTF8.GetBytes($pw))|%{$_.ToString("x2")}) -join "")
     ```
3. 実カードデータを取得（番号は pokemon-card.com の `details.php/card/<番号>`）:
   ```powershell
   powershell -ExecutionPolicy Bypass -File tools\scrape-cards.ps1 -Numbers 50220,50221 -Out js\data\cards.local.js
   ```
   生成された `js/data/cards.local.js` も `.gitignore` 済みで公開されません。
   デッキは未定義なら手持ちカードから自動構築されます（`js/data/decks.local.js` で番号指定も可）。

### デッキコードから丸ごと取り込み
pokemon-card.com のデッキコード（`deck/confirm.html/deckID/<コード>`）から、デッキ内の全カード（データ・画像・枚数）を取り込めます:
```powershell
powershell -ExecutionPolicy Bypass -File tools\import-deck.ps1 -DeckCode ppRySR-eJ2k2F-p3ypXS
```
`cards.local.js`（全カード）と `decks.local.js`（番号と枚数のデッキ定義）が生成され、メニューに「取り込みデッキ」が追加されます。ポケモンは戦闘データ（進化チェーン含む）を完全取得。

主要トレーナーズは効果を実装済み：**ボスの指令・ふしぎなアメ・なかよしポフィン・ポケパッド・夜のタンカ・せいなるはい・スイレンのお世話・トウコ・ヒカリ・ビワ・ワンダーパッチ・改造ハンマー**（山札/トラッシュ操作はカード選択モーダル、相手への効果は対象クリックで指定）。

**スタジアム**にも対応（1ターン1枚・同名不可・別スタジアムは置換してトラッシュ）。バトルコロシアムの「相手の効果でベンチにダメカンがのらない」も実装。

**特性（とくせい）**も実装：誘発（サイコドロー＝進化時にドロー）、起動（にげあしドロー＝3枚引き自身を山札へ／さかてにとる＝前の相手の番にきぜつしていれば3枚引く・自分のポケモンをクリックして起動）、継続（はなのカーテン＝相手の効果でベンチにダメージを受けない）。未対応のトレーナー/特性は「出せるが効果なし」、特殊エネルギーはタイプ推定で付与されます。

**AIも人間と同等**に行動します。「取り込みデッキ」「マイデッキ」を選ぶと AI も同じ実デッキでミラー戦になり、実カードのトレーナーズ・スタジアム・特性（ボスの指令でのKO狙い、ふしぎなアメ、サーチ系サポート、ドロー特性など）を自動で使います。

AIの戦術は **弱点・抵抗を加味した実効ダメージ**でワザ・引きずり出し対象・昇格・入れ替えを判断し、エネルギーは「そのターンにワザを撃てるようになる」付け方を優先、ドロー特性やサーチサポートの乱用（自滅的な山札消費）も抑制します。

攻撃ワザ効果も拡充：「手札の枚数ぶんのダメカン」（フーディンのハンドパワー）、**「相手のポケモン1匹に○ダメージ」＝対象選択**（キチキギスexのクルーエルアロー。人間はダメージを与える相手をクリックで選択、ベンチは弱点・抵抗を計算しない）、「このポケモンをベンチと入れ替える」（テレポートアタック等）。

> 画像はリンク参照のみで、ダウンロード・再配布はしません。あくまで個人のローカル利用に限定してください。

## テスト

依存なしのブラウザ用テストランナー。HTTP 配信して `tests/` を開くだけ。

```
# 例: ローカルサーバを立てて
http://localhost:8790/tests/
```

エンジン（弱点/抵抗/コスト/きぜつ/サイド/先攻制限）、AI 対 AI 通し（決定的シードで15戦）、パスワード錠、自動デッキ構築を検証します。結果は画面と `window.__TEST_RESULTS` に出力。
